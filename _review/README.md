# 开发复盘 & 踩坑记录

记录刷题过程中的真实问题——比"做完了"更值钱的是"做错过"。

## 结构

```
_review/
├── README.md              # 本文件，索引
├── pitfalls.md            # ★ 滚动累积的"踩坑清单"（按主题分类，方便日后查找）
└── <题号>_<短名>.md       # 具体某题的问题清单（每道遇到坑的题一份）
```

两种维度互补：
- **横向（pitfalls.md）**：按"主题"累积。比如所有"Python 字符串不可变"的踩点都归入字符串那一节。日后写新题前扫一眼就能避坑。
- **纵向（按题）**：按"题"复盘。某题做完，把那次的具体 bug 列出来；半年后回看这道题时能秒回忆"我当时栽在哪"。

## 用法

每次写题遇到问题（语法报错、逻辑卡壳、运行结果不对），按下面流程：

1. **当下** — 在该题的 `<题号>_<短名>.md` 里，简记三件事：
   - 错误现象（拷贝报错信息一行）
   - 根因（一句话）
   - 怎么改的
2. **当周** — 周末复盘时把当周新的根因归类抄进 `pitfalls.md` 对应主题下；如果该主题不存在就新开一节
3. **复盘** — 翻 `pitfalls.md` 比翻原题文件高效得多，因为同一类错误你会犯好几次，集中归档后能识别模式
